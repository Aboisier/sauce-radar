Sauce radar

I'm sorry what

Got tired of commenting the same stuff on pull requests, so I made this. It finds stuff in PRs using regular expressions. I know, it's pretty limited, but it makes it language agnostic, so that's that, still pretty cool. Sauce radar.

Setup

First of all, install the app.

Then, create a config file named sauceradar.yml in the .github directory of your repo. 

  # List of regexes. The rule will be applied to files matching one of these regexes.
- files: 
  - .*\.js

  # List of regexes. The rule will be applied for PRs whose target branches match one of these regexes
  branches: 
  - develop
  - feature\/*
  
  # Regex sauce radare uses to detect sauce. The rule is tested per-line
  rule: function ([^\s(]*)

  # Multiline or single-line comment that the app will comment if the rule matches
  # You can use capture groups from the rule, e.g. Wow, {0} is a great function name!
  comment: Woo

Final words

Well that's pretty much it. From now on, every time a PR is created or updated on your repository, the sauce radar will test the changes against the rules and post comments whenever a rule matches.
